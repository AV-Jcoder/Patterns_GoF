По аналогии с Фабричным методом, Абстрактная фабрика нужна,
чтобы вынести весь код, где инстанцируются объекты
в отдельный класс или модуль.

Но в отличии от Фабричного метода, который инстанцирует
какой-то один объект, Абстрактная фабрика способна
создавать группу взаимосвязанных объектов.

Таким образом можно предоставлять полльзователю
библиотеку объектов, раскрывая только их интерфейсы
а не реализацию.

Итого :
1. Фабрика изолирует конкретные классы.
Клиенты манипулируют экземплярами через
их абстрактные интерфейсы.

2. Упрощает замену семейства продуктов.
В коде мы инстанцируем Абстрактную фабрику
с помощью new только один раз, а значит можем легко
сменить одну реализацию на другую и всё продолжит работать!

3. Гарантирует сочетаемость продуктов.
Допустим window и scrollBar длжны работать вместе.
И при реализации фабрики мы должны гарантировать, что
возвращаемые её методами обьекты будут совместимы.

4. Трудно модифицировать.
Интерфейс фиксирует набор объектов, которые можно создать.
А для тогшо чтобы добавить еще несколько объектов, которые
иннтерфейс может создавать его нужно модифицировать,
а также все его подклассы.
