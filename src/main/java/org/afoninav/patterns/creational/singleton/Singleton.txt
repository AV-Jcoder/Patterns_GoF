Данный шаблон гарантирует, что у класса
будет только единственный экзземпляр и
к нему будет предоставлена глобальная
точка доступа.

Реализзация подраззумевает private конструктор
и static метод, где инстанцируется единственный экземпляр.

В ззависимости от реализации могут быть
проблемы в многопоточных приложениях,

Обычно для многопоточных сред применяется вариант
с двойной проверкой и одной блокировкой.

Единственный экземпляр наследутся в подклассах.

Синглтон применяется, когда нам нужен один единственный объект,
например для укравления каким либо каркасом существующих объектов.
